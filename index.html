    For Developers

    Wrtie a media player using QtAV is quite easy. QtAV provide both Qt C++ and QML api. QML playback API is designed to compatible with QtMultiMedia

    Here is a minimal player example. For more detail to using QtAV, see the wiki Use QtAV In Your Project and examples

    C++ QML 
    WidgetRenderer renderer;
renderer.show();
AVPlayer player;
player.setRenderer(&renderer);
player.play("test.avi");

    import QtQuick 2.0
import QtAV 1.3
Item {
    VideoOutput {
        anchors.fill: parent
        source: player
    }
    AVPlayer { //or MediaPlayer
        id: player
        source: "test.mp4"
    }
    MouseArea {
        anchors.fill: parent
        onClicked: player.play()
    }
}
    The required libraries for building QtAV I use can be found here: FFmpeg and OpenAL, Qt

    Features

    Supports most playback functions. Play, pause, speed, seek, subtitle, volume, audio channel, audio track, dynamically renderer switching, video equalizer, custom filters for both window and video frame, network stream, video crop, multiple renderers, hardware decoding...

    With the help of FFmpeg, QtAV is able to play most media formats, without installing other codecs.

    Hardware decoding

    Supports DXVA, VAAPI, CUDA, VDA, CedarX. Direct rendering is also supported for VAAPI and it gives you better performance. Direct rendering will be support for DXVA, CUDA and VDA in the future.

    CUDA for Windows and Linux

    Decoding with CUDA is supported since 1.3.3. Tested on Windows and Linux. It's the first player/library supports CUDA on Linux(tell me if i'm wrong)

    Hi10P support by CUDA

    Now some NVIDIA cards support Hi10P. I tested GT 540M and GT 750M. So you can enable CUDA decoder in QtAV and get better performance.

    OpenGL and ES2 support for Hi10P

    Rendering 16-bit YUV is supported using OpenGL and OpenGL ES2. Supported formats are 9, 10, 12, 14, 16 bit planar YUV, both little endian and big endian. AFAIK, those famous player such as VLC, XBMC and mplayer can not renderering it using OpenGL ES2. They only support desktop OpenGL. Obviously QtAV has better performance. I tested a 10bit h264 video (download here: http://ps-auxw.de/10bit-h264-sample) on HTC Butterfly S. QtAV cpu usage is about 10% less than VLC!

    Performance

    QtAV supports hardware decoding and OpenGL renderering. The performance is quite good. You can compare QtAV with VLC, XBMC and MPV.

    Platforms

    Currently QtAV can run on Windows, Linux, Mac OSX, Android, iOS(works fine on simulator), MeeGo, Sailfish OS.

    Other platforms such as WinRT, Windows Phone, BlackBerry will also be supported in the feature, but I need development environment and devices.

    Why Not QtMultimedia? Why QtAV?

    • QtMultimedia in Qt4 is poor. QtAV works fine for both Qt4 and Qt5.
    • QtMultimedia's implemention is platform depended. It use dshow on windows, gstream on linux etc. May be not an easy work to support a new platform. While QtAV use FFmpeg, works for most platforms. No additional work required.
    • Additional codecs may be required to make QtMultimedia work for some media formats.
    • QtAV has more powerful playback features than QtMultimedia. For example, multiple renderers for 1 player, region of interest(video crop), custom filters
